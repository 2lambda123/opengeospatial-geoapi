Changes from GeoAPI 1.0 to GeoAPI 1.1

  • In CRSFactory.createImageCRS(...) method, changed the cs argument type from CoordinateSystem to ObliqueCartesianCS. This change is required for type safety and was forgotten in previous release.

  • In CSFactory interface, added methods for UserDefinedCS creation.

  • In MathTransformFactory interface, added createAffineTransform(AffineTransform) convenience method for interoperability with Java2D.

  • Renamed NoSuchClassificationException as NoSuchIdentifierException.

  • In Envelope interface, added convenience methods getMinimum, getMaximum, getCenter and getLength.

  • For convenience method getParameter(String) in OperationParameterGroup interface, changed the return type from GeneralOperationParameter to OperationParameter. This change make the convenience method more... convenient.

  • For convenience method getValue(String) in ParameterValueGroup interface, changed the return type from GeneralParameterValue to ParameterValue. This change make the convenience method more... convenient.

  • In SampleDimension interface, added a Locale argument for methods getDescription and getCategoryNames. This change is required for localization support and was forgotten in previous release.

  • In Coverage interface, added a Locale argument for method getDimensionNames. This change is required for localization support and was forgotten in previous release.

  • In Coverage interface, changed the return type from double[] to float[] for the method evaluate(DirectPosition, float[]). Previous return type was wrong.

  • In Coverage, merged getNumSources() and getSource(int) methods into a single getSources() method.

  • In GridRange, renamed getLower() and getUpper() as getLowers() and getUppers().

  • In GridRange, added getLower(int), getUpper(int), getLength(int) and getDimension convenience methods.

  • Javadoc fixes.
